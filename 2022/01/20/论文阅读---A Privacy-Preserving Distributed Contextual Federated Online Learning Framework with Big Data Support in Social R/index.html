摘要

如今，大数据分析的蓬勃发展以及计算能力和网络带宽的限制，使得独立的智能体/服务提供商很难在有限的时间内从大量的在线数据中为每个用户提供合适的信息。为了应对这一挑战，推荐系统（RS）可以调用一组智能体来协作学习用户的偏好和口味，这就是所谓的分布式推荐系统（DRS）。DRSs可以通过要求智能体相互分享信息来提高传统RS的准确性。然而，由于有大量的候选者，DRSs为每个用户做出个性化的推荐是具有挑战性的。此外，智能体之间的信息共享引起了隐私问题。因此，我们在本文中提出了一个保护隐私的DRS，然后将每个服务提供者建模为一个具有上下文意识的分布式在线学习者。服务提供者通过合作，根据用户环境和用户的历史行为，学习用户的偏好，做出个性化的推荐。我们采用联邦学习框架来帮助在大量的分布式智能体上训练一个高质量的隐私保护的集中式模型，这在相对缓慢的网络连接中可能是不可靠的。为了处理大数据场景，我们建立了一个项目集群树来处理在线和从上到下不断增加的数据集。我们进一步考虑了社会网络的结构，并提出了一种有效的算法，以适应性地避免更多的性能损失。理论证明表明，我们提出的算法可以为服务提供商和用户同时实现亚线性遗憾和差分隐私保护。数值结果证实，我们的新框架可以处理越来越大的数据集，并在隐私保护水平和预测精度之间取得平衡。

介绍

一个推荐系统（RS）可以理解用户的喜好并向他们推荐理想的物品。由于用户的品味随模式而变化，人类倾向于接受与他们之前表现出的兴趣相似的物品，以及其他类似行为的人喜欢的物品。因此，网络上关于物品（产品、设备、事件和服务）的评论、评级、评论、等级、投诉、意见、索赔和特征等形式的大量增加的数据可以被用来做出正确的决定和推荐。这些信息通常包含来自不同用户的历史反馈以及项目和用户之间的相关相似性。如果公布出来，对大数据分析师设计有效的数据挖掘和机器学习方法极为有利。然而，这些数据通常是敏感和私密的，直接公布可能会对居住在RS的用户和服务提供者的安全造成威胁。为了在数据信息的合理使用和用户的隐私之间取得平衡，政策制定者必须运用隐私法的一些最基本的概念，这对于发展RS的大数据技术特别重要。

在大数据时代，数据信息通常是海量的、异质的，并随着时间的推移从不同的来源演变。术语 "大数据 "通常由4V维度显示。数量、种类、速度和真实性[3]。在一个RS中，容量(Volume)是由用于生成建议的海量数据来编码的。（量大）种类(Variety)是指从不同的来源（如博客、Facebook、Twitter）提取的数据。（种类多）速度(Velocity)代表了在线生成数据的速度。（速度慢）由于数据量大，RS逐渐变得难以为用户找到有用的、合适的信息，其他三个维度也是如此。因此，一个具有大数据支持的RS，旨在帮助用户从大量的备选方案中做出有益的选择是非常可取的。此外，由于大数据的多样性，很难满足所有类型的人的口味。因此，一个RS应该能够为不同类型的人提供个性化的推荐。此外，考虑到大数据的速度，大量数据造成的计算和带宽限制使一个独立的服务提供商无法获得所有有用的信息（例如，其竞争服务提供商所拥有的信息）。因此，为了促进总体表现，服务提供商更愿意获得其他服务提供商共享的低价信息。我们可以建立一个由服务提供商（智能体）组成的在线社会网络来实现这一目标，因为在线社会网络可以让大规模的不同智能体进行合作和协调[4]。这种RSs被称为分布式推荐系统（DRSs）[5]，一组智能体合作发现相关信息并预测用户行为。然而，对DRSs的研究是一个新兴的、不成熟的领域。为了激发我们的想法，我们提供了一个引人注目的、即将到来的现实应用，如下。

移动边缘计算（MEC）作为一种新兴技术，允许将计算服务传送到由移动基站控制的移动社交网络中的边缘节点[6]。 MEC的架构构成了地理分布和互连的边缘节点，配备了服务器和存储单元。移动用户会将繁重的计算任务卸载到边缘节点。用户主要向边缘节点发送编码为上下文的服务请求。他们对社交和媒体服务的频繁请求需要边缘节点中的缓存功能。因此，DRS 本质上是必要的。边缘节点支持将新的社交和媒体数据（项目）上传到其存储单元。隐私也是边缘节点和移动用户关注的核心问题。此外，由中央可信系统控制的边缘节点之间共享大量信息，确保设计的分布式系统的安全保护。为了解决所有这些问题，[43] 中的作者指出，联邦学习可以用作边缘计算的操作系统。

总之，DRS 有望提高推荐性能。然而，正如我们在相关工作部分中看到的，很少有数据挖掘和机器学习算法能够完全支持大数据的 4V 维度。此外，隐私问题现在正成为分布式系统广泛关注的问题（例如，[7]、[8]）。因此，它是DRS的一个迫切研究课题。一方面，智能体商可以共享信息以互惠互利。在线社交网络中的智能体可以是任何可以提供有用服务的个人或公司。例如，在产品销售系统中（例如，亚马逊和淘宝），不同的公司可以互相帮助销售产品。智能体通过这种合作可以获得更大的回报。另一方面，直接发布此类敏感信息会引起智能体及其用户的隐私问题。因此，我们有必要考虑两种不同的隐私泄露：用户的背景信息（上下文）和智能体的项目存储库。这里的挑战可以总结为：如何构建具有大数据支持的可以进行个性化推荐的DRS，以及如何在不泄露个人和智能体隐私的情况下保证信息的实用性。

为了解决上述问题，我们设计了一个基于树状结构的上下文分布式在线和隐私保护的大数据处理框架，其中大多数连接的智能体可以通过部署隐私保护机制，在信息共享网络上进行个性化的预测。然后，我们让一组智能体通过一个固定的网络相互连接。而每个智能体都有用户流入其服务器/平台的经历。一旦智能体收到某个用户的到来，它就会从自己的数据库中推荐一个项目给该用户。在技术上，我们提出的社会DRS是基于上下文多臂匪徒（CMAB）[46]。CMAB通过设置一个相关的变量来衡量每个手臂（或项目）的不确定性水平和估计性能，该变量根据历史记录产生，并在每轮更新。然后，它根据所选项目的历史反馈，在每一轮中选择具有最高奖励的项目。

相关工作

我们的工作属于社交推荐系统。一般来说，RS 部署内容过滤方法或协同过滤方法。内容过滤方法为每个用户和项目建立一个配置文件。然而，这种配置文件可能难以收集。协同过滤方法仅依赖于过去的用户行为，当历史记录很少时，这会导致冷启动问题。

如今，大数据分析技术的匮乏成为RS面临的又一挑战。基于用户社交资料和关系的代表性传统 RS 在中。然而，几乎所有这些都不支持用户和项目类别中的超大数据集。尽管 [25] 中显示了最近的此类解决方案，但它不支持动态传入的在线数据和更改用户的上下文。所有这些挑战使得经典推荐方法在许多实际场景中不可行。

多臂老虎机 (MAB) 方法作为一种有前途的解决方案，可以在 RS 中实现重要的勘探-开发权衡以提高性能。在 MAB 算法上已经完成了大量工作。他们中的一些人专注于 DRS。例如，布卡帕特南等人。通过利用带有 UCB1 的在线学习方法提出了一种 RS，其中智能体可以相互共享以提高对固定大小和小规模数据集进行预测的准确性。在大数据分析中应用 MAB 的识别源于工作，其中提出的 UCT 算法基于 UCB1 和蒙特卡洛树搜索（MCTS），它可以应对潜在的无限多臂。作品是具有代表性的进展。然而，它们都不是上下文感知的，这限制了它们在 RS 中的应用。

许多以前的作品都讨论了CMAB。例如，Cem 等人引入了一个 DRS 来实现基于个人需求的预测，并且具有固定大小的项目集。然而，所提出的上下文划分方法是静态的，如果它在大规模数据集中工作，将导致巨大的计算复杂性。因此，对于大数据分析，显然是不切实际的。所以宋等人。引入了一个 RS，它可以为单个智能体自适应地划分上下文 [26]。但是，item-cluster树是从下到上构建的，完全限制了item的数量。因此，它也不能处理大数据分析。单一智能体RS[26]不能处理来自分散的或多媒体资源的大数据。

在我们的问题中，我们采用了 MCTS 的基本思想，它可以用来分析项目子空间而不是单个项目，以解决不断增加（并接近无限）项目空间大小的问题。为了处理大数据场景，我们提出了一种项目集群树结构，可以随着时间的推移从上到下而不是从下到上自适应地扩展这些项目集群[26]。我们的自上而下的结构支持对大规模和不断增加的数据集的推荐，因为所有智能体都在项目集群级别而不是单个项目级别上提出建议 [26]。因此，树中的组件对于所有实现大数据分析的智能体都是可扩展的。此外，由于动态扩展的项目簇树，我们的方法可以支持在实际应用中增加数据集，而其他 [18]、[22]、[23]、[24]、[26]、[28]、[29 ]，[39] 不能。我们可以将每一轮构建的覆盖树中子节点的选择建模为独立的 MAB 问题。然而，与 [11]、[12]、[13] 相比，我们在具有上下文感知的 CMAB 模型中设计了一个自适应上下文空间分区模型。它随着时间的推移而发展，以保证智能体可以根据具有最相似上下文的先前到达用户的相应数据来探索当前到达用户的上下文。因此，它提供了最新的个性化项目推荐。此外，我们在我们的 CMAB 模型中利用 [49] 中的阈值技术（详见第 3.1 节）来获得迄今为止最好的在线学习后悔性能。此外，我们的算法是一个分布式框架，探索了智能体合作和网络结构（详见第 4 节），以极大地提高 DRS 的学习性能。

当谈到 DRS 中的隐私保护问题时，匿名性的应用是广泛而有用的 [34]。但事实上，常见的匿名算法可能会影响大数据在高维 [24]、[32]、[33] 中的潜在效用。此外，由于匿名性，一旦对手相互勾结或辅助信息可访问，用户将被重新确认[31]。此外，一些现有的算法经常利用密码学 [37] 来确保隐私不受攻击。虽然这些方法可以保证安全性，但它们往往会导致高昂的通信成本和计算复杂性，从而极大地影响大数据分析。差分隐私（DP）作为一种很有前途的技术，可以解决隐私问题[9]，已在多项研究中用于 RS。 DP中不需要对手的背景信息。然后 DP 只获取数据库管理器给出的相关查询结果，对单个数据条目的更改（删除或插入）不敏感。也就是说，当用户发布的数据满足DP，而其他攻击者使用一些帮助数据分析的工具请求数据库管理器时，即使用户的记录不存储在本地，请求结果也几乎相似。因此，无法推断用户的隐私，并且 DP 在大规模数据集 [10] 中应用时对预测精度的影响很小。因此，将 DP 与 BDA 方法一起应用可以保证隐私和预测准确性。在本文中，我们采用 DP 作为我们的隐私保护策略。在 [39] 中，DP 被应用于推荐社交图中的任务。但是[39]考虑了社交网络中智能体之间的社交关系和用户偏好的私人信息，而我们则关注用户的背景和智能体的物品库的隐私问题。

为了获得可扩展且可靠的分布式解决方案，我们在我们的设置中采用联邦学习框架 [42] 来有效地训练集中式模型，当训练数据分布在许多具有相对缓慢和不可靠的网络连接的智能体上时。联邦学习是“通用数据保护条例”[44]下人工智能研究中一个非常近期且有前途的研究方向，其详细概念和应用见[43]。它的主要思想是在云或受信任的第三方 (TTP)（或策展人）上保留一个集中式模型，并且每个本地智能体都会使用隐私保护通信方案将更改汇总为对策展人（第三方）的小型集中更新。同时，curator（TTP）立即将更改与其他智能体的更新进行平均，以促进共享模型。所有训练数据都保留在本地智能体上，没有更新单独存储在策展人中。

在本文中，我们应用 DP 为从位于每个智能体处的用户数据训练的模型提供整体隐私保证。使用联邦学习的思想，每个智能体将部署一个拉普拉斯机制来在本地训练历史记录[35]。当来自其单跳邻居的共享信息查询到达时，TTP 将请求转发给他们并通过部署**指数机制来保持隐私 **[36]。 TTP 只保留一个整体的隐私保护树型学习模型，而对 TTP 的整体预测是通过结合这个局部模型“集合”的各个预测来生成的。因此，它是针对大规模分布式网络场景的可扩展解决方案。与传统方法通过向每个相关记录添加噪声会导致很大的失真不同，我们引入了一种基于自适应二叉树的噪声聚合方法来保证 DP 并同时避免性能损失。此外，智能体通过指数机制根据用户的偏好提供在线学习结果。此外，我们的theistical证明证实了我们的方法可以确保用户敏感的个性化上下文和智能体共享信息的 DP。此外，我们还考虑了网络结构的影响。

贡献

1.我们提出了一种保护隐私的联邦在线学习算法，该算法支持大数据的DRSs（分布式推荐系统）的上下文感知。具体来说，大数据 "4V "维度中的 “真实性”、"多样性 "和 “速度”（除了基本的 "体积 "特征）以及社会化DRS（分布式推荐系统）中的 "隐私 "这一关键问题，在我们提出的框架中首次作为一个整体解决方案得到了很好的解决。

2.DRS对用户的情境空间进行了自适应的划分，以做出个性化的推荐。数值实验结果表明，我们的建议可以支持现实世界中不断增加的大数据集，并优于其他最先进的隐私保护方案和在线学习方法。

3.通过将严格的攻击模型以及指数和拉普拉斯机制定义为DRS，我们的算法可以收敛到最佳策略，并保证在一段时间内保护隐私。

4.我们在保护隐私的DRS中引入了联邦学习，它只逐步更新驻扎在TTP中的一个集中式模型。它提供了更智能的模型，更少的功耗和更低的延迟，并确保了隐私，这适用于不可靠的和相对缓慢的网络连接，具有极好的可扩展性。

5.通过对网络拓扑结构的探索，它是一种具有在线学习社会网络结构的自适应算法，可以加快学习速度，广泛地减少性能损失。

结论

我们提出了一种新的分布式联合在线学习算法，该算法具有上下文感知和大数据支持，用于社会DRS，并设计了一个框架来适应性地提高预测性能。然后，我们给出了详细的理论分析。实验结果表明，我们提出的算法可以实现以下目标：合适的物品推荐，对用户偏好的准确预测，对用户情境的差异化隐私保护，以及对智能体库的差异化隐私保护。
